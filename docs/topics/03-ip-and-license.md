## 所有権（対話上のFAQ記録）

| 項目 | 対話内の要約 | 種別 |
|------|----------------|------|
| OpenAIへの著作権譲渡 | **ない**と説明されている | [FAQ記録・公式再確認要] |
| 残る権利 | 開発した個人／チーム／組織に IP が残る | [FAQ記録・公式再確認要] |
| OpenAIが得るもの | 審査のための非独占ライセンス（所有権ではない）と説明 | [FAQ記録・公式再確認要] |
| 広報 | 氏名・肖像・声・画像等をハッカソン期間＋その後3年利用可との説明 | [FAQ記録・公式再確認要] |
| 応募者保証 | 提出物はオリジナルで単独所有、第三者権利侵害なし | [FAQ記録・公式再確認要] |
| OSS | 利用可だがライセンス遵守＋元製品への機能的拡張が必要 | [FAQ記録・公式再確認要] |

### 対話の核心フレーミング

> 「権利を取られる」より「**完全に自分が所有していると保証できない**」方が現実リスク。

---

## 持ち込んではいけないもの

無断データセット / 規約違反スクレイピング / 顧客実データ / 社内コード / 他人の過去PJ / ライセンス不明テンプレ / 著作権音楽 / 無許諾ロゴ・商標 / 顔・音声・個人情報 / 職務著作・会社帰属発明 / 共同開発の権利未整理 / 外注コードの著作権未譲渡 / 投資家・企業が権利を持つ既存PJ

---

## Public vs Private / ライセンス選択

| 方針 | 内容 | 種別 |
|------|------|------|
| 開発中 | Private・ライセンスなしで開始 | [戦略] |
| 提出時 Public | 審査アクセス事故回避に有利。Apache-2.0を選ぶ案（MITより特許条項が明示的） | [戦略。規約要件ではない] |
| 事業化温存 | Privateのまま審査メール共有。OSS義務なしという対話内整理 | [FAQ記録 + 戦略] |
| ハイブリッド | 本体保持、SDK／スキーマ／fixtureだけ後日Apacheで切り出し | [戦略] |
| 注意 | Public無ライセンスは避ける | [戦略] |
| Apache後 | 一度配布した版の許諾を後から独占へ戻せない | [一般論] |

## アイデア vs 表現

著作権は主にコード・文章・画像等の**表現**。アイデア・機能コンセプト・BMは通常保護されない。公開デモ後の模倣リスクは譲渡条項とは別問題。特許候補は公開前に判断 [一般論]。

---

## デモデータの権利（探索系をやる場合）

第三者SDK/API/データは利用条件遵守。実在企業LPを画面いっぱいに映すのは商標リスク。

安全策: **自作架空LP／自サイト／許可サイト／匿名化サンプル**。無差別クロール禁止設計。
