# Flagship reliability capture notes

These captures were generated by
`tests/e2e/flagship-reliability-evidence.spec.ts` with
`CAPTURE_EVIDENCE=1`. Every scenario resets the seeded Flagship meeting before
and after capture.

All content, credentials, WebRTC peers, SDP exchanges, provider responses, and
regulatory events are synthetic. Decision synthesis uses the standard
Playwright server's deterministic fake-provider mode. No real API key,
ephemeral secret, microphone audio, provider call, customer content, or private
user data is present.
