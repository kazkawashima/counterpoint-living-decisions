# Active goal — Descant demo and public-safety boundary

更新日: 2026-07-21
状態: ACTIVE
優先度: submission-critical

## 目的

ハッカソン提出に向け、今回の作業対象を次の3点に限定する。

1. 公開表示名を `Counterpoint` から `Descant` へ変更する。
2. 審査員が迷わず Flagship を再生できる、一本の合成デモ導線を完成させる。
3. リポジトリを公開し、審査用 server-funded AI を本番で有効化する前に、
   秘密情報・個人情報・予算超過・権限越境を防ぐ公開安全境界を証明する。

提出用の表示名は当面 **Descant — Living Decisions** とする。リポジトリ名、
パッケージ名、`@counterpoint/*` の内部 import、D1/R2/Worker の既存リソース名、
DB上の識別子は移行リスクを避けるため変更しない。歴史資料に残る旧称は、現在の
公開プロダクト名と誤認されないよう扱いを分ける。

## 調査で確認した現在地

- Flagship は合成データ、固定デモユーザー、`Context → Permission →
  Commitment → Risk → Review` の5段階、staged event、facilitator resetを持つ。
- OpenAI と judge のCloudflare経路は preview では disabled。したがって現在の
  preview は手動テキスト経路を含む安全な degraded mode として扱う。
- Worker側には judge allowlist、別HMACによるIP擬似化、D1予約台帳、opaque call
  handle、rolling 24時間 USD 25 hard cap、`USAGE_LIMIT_REACHED` の基盤がある。
- providerのUSD 50 alertは二次警告であり、アプリ側のUSD 25制限の代替ではない。
- C5の hosted Worker security matrix 再実行、C6の実デプロイ記録・最終D1/DO確認、
  judge secret登録は未完了。production judge を先に有効化しない。
- 直近の hosted parity 修正により、meeting作成、decision history、audit、export
  のWorker経路は実装済み。表示専用tokenの発行・projection・失効もWorkerへ接続済み。
  最新の名称・導線 preview は commit `221b6d5` で、
  `https://counterpoint-living-decisions-preview.gs2safari.workers.dev` にデプロイ済み。
  旧 `counterpoint-living-decisions.gs2safari.workers.dev` は別Workerで旧称のままなので、
  審査導線には使用しない。
- `docs/思うところつらうら/動作確認後に修正したいこと.md` は検討材料として読むが、
  ユーザー保護のためこのgoal以外の作業で編集・公開しない。
- `221b6d5` preview deployでは security matrix 300/300、Cloudflare pool 142/142、
  health/readiness/unauthenticated auth smoke、Flagship smokeを通過した。実previewの
  read-only probeでも、logged-out root/health/readyは200、認証前APIは401、通常ユーザーの
  judge usageは503 `REALTIME_UNAVAILABLE`、display projectionは200、失効後は401
  `DISPLAY_TOKEN_EXPIRED`、projectionにprivateWorkspace/private textは出ないことを確認した。
- previewではAI経路をdisabledのまま、Flagship meetingに限る provider-free の
  `staged-demo-rule-v1` を明示的な `DEMO_STORY_MODE=enabled` として注入した。これは
  secret、provider call、usage reservationを持たず、production用render configでは強制的に
  disabledになる。したがってpreviewのstaged demoは安全な合成導線であり、production
  judgeを有効化したことを意味しない。
- READMEとjudge credential運用手順は、固定ログイン情報を漏洩しても本番権限にならない
  synthetic local/demo fixtureとして明示し、本番judge credentialのrevoke・rotation・
  Secret delete手順を記録している。ローカル/Preview E2Eはprovider disabled、provider
  failure、usage capのいずれでも手動入力、状態、history/audit/exportが継続することを
  確認している。
- Previewの明示デプロイを、commit、config hash、migration境界、Worker/R2/DO binding、
  safety flags、検証結果を含むcredential-free記録
  `docs/deployments/preview-2026-07-21.md` に残した。productionは未デプロイ・未有効化である。
- `221b6d5` preview deployでは security matrix 300/300、Cloudflare pool 142/142、
  health/readiness/auth smoke、Flagship smokeを通過し、実previewのPlaywrightでも
  `AT_RISK → facilitator review → REVIEW_REQUIRED → Action hold/reconsideration task`
  とリロード後の継続、ordinary/judge/displayの別browser context境界を確認した。
- 2026-07-21の外部host Preview Playwright再実行は3/3ケース成功、41.8秒だった。
  これはprovider-free staged review/manual fallback、SPA/API同一origin、
  ordinary/judge/display境界のpath-health証拠であり、初見の人間による3分リハーサルとは
  分けて扱う。

## 完了条件 B — 一本の審査デモ導線

### 審査員の開始点

- [x] ログアウト状態の公開URLから、最初に表示される画面だけで「これは合成データの
      staged hackathon demo」であること、次に押す場所、AIが使えない場合の手動経路が
      分かる。
- [x] 固定の合成ユーザーでログイン後、room作成・参加者割り当てを要求せず、seed済み
      Work & Productivity Flagship を一覧から開ける。
- [ ] judge credentialは公開README、動画、リポジトリ、Devpostの公開欄へ置かず、
      非公開と確認できた審査用Testing Instructions等の経路だけで渡す。

### 導線の価値弧

- [x] facilitator一人（必要なら別タブの合成participant）で、次の順序を画面上で追える。
      `Private context → exact disclosure preview → owner approval → shared Evidence
      → human Decision commit → staged external event → AT_RISK → human review →
      REVIEW_REQUIRED / Action hold / reconsideration task`。
- [x] 各段階に一つの主操作と短い説明があり、初見でも次の操作を推測できる。
- [x] `Private`、`Shared`、`AI inferred`、`Human confirmed`、`Staged demo story` を
      混同しない表示にする。staged eventは実世界の連続監視・実在規制通知ではないと
      明示する。
- [x] `Commit` 後に history、audit、exportを取得しても、未接続経路を artifact storage
      エラーとして誤表示しない。API失敗時は対象機能、原因、再試行または手動継続を示す。
- [x] OpenAI disabled、provider障害、judge cap到達のいずれでも、既存のroom状態、
      手動excerpt、Decision編集、history、JSON exportを保持する。
- [x] Flagship resetは対象meetingだけを初期状態へ戻し、再実行して同じ価値弧を再生できる。
- [ ] 初見の通し実演を約3分以内に収める。機能追加で長くなった場合は任意機能を導線から
      外し、Flagshipを優先する。

### UI検証と記録

- [x] UIを変更した各sliceで、対応するPlaywright browser E2Eを追加・更新し、実行結果を
      commitする。
- [x] `localhost`だけでなく、Cloudflare previewまたはTailscale相当の外部hostで、SPA、
      same-origin API、artifact、必要な realtime URLにCORS/localhost固定の失敗がない。
- [x] desktop、mobile、reduced-motionの主要状態を合成データだけでスクリーンショット
      に残す。reel制作自体はこのgoalの完了条件に含めない。

- [x] `.env.example` は空のplaceholderと安全な説明だけを持ち、judge keyは記載しない。

- [x] 固定デモパスワードは実アカウントの認証情報ではなく合成demo-only credential
      と明示し、seed dataもすべて架空にする。公開後の漏洩を前提に、judge credentialは

- [ ] 直接依存・推移依存・同梱mediaのlicense/NOTICEを棚卸しし、公開切替前に
      `MIT` または `Apache-2.0` 等の最終方針を決める。未確定のまま公開しない。

### 認証・権限・AI費用

- [x] previewでは `OPENAI_MODE=disabled`、judge structured/realtime route disabledを
      維持し、未設定・障害時は fail-closed しながら手動経路を残す。
- [ ] productionでjudge modeを有効にする場合、exact allowlistのjudge user、正規化済み
      `CF-Connecting-IP`、別HMAC secret、全feature flag、production bindingを同時に
      満たさない限り利用できない。
- [ ] judgeはserver-fundedでBYOK不要、普通のユーザーはjudge keyを選択・継承・利用
      できず、browser response/network/logにprovider credentialやprovider call IDが出ない。
- [ ] meeting、account、IP pseudonym、concurrency、Realtime seconds、generation、
      tokens、rolling 24時間costの全制限をbillable workの前に確認する。
- [ ] rolling 24時間 USD 25をアプリ側hard capとして強制し、USD 50 provider alertは
      二次警告として扱う。どの制限到達時も新規provider requestを出さず、明示的な
      `USAGE_LIMIT_REACHED` と手動継続を返す。
- [ ] managed realtime/structured AIの各routeについて、同一リクエストの再送、並行開始、
      meeting越境、session/owner差替え、終了競合をテストし、二重課金しない。

### 公開前の証明

- [ ] security matrixをhosted Worker経路でも再実行し、IDOR、meeting isolation、

- [x] logged-out browserでroot/health/ready/auth boundaryを確認し、private dataや
      judge-only情報が認証前に見えない。
- [x] ordinary user、judge user、display tokenを別browser contextで試し、権限境界と

- [ ] preview/productionの対象commit、config hash、migration状態、Worker/R2/D1/DO

- [ ] 本番secretを登録する前に、rollback、judge shutdown、credential rotation/deleteを
      実行可能な手順として確認する。

## 完了の定義

このgoalは、名称変更後のFlagshipを外部hostのclean browserで再生でき、AIが
disabledでも手動経路で価値弧が完走し、hosted security/secret/cost/authorizationの
証拠が揃い、README・提出説明・testing instructions・license方針が一致した時点で
ACHIEVEDとする。スクリーンショットやreel素材の記録は開発規律として継続するが、
reel完成はこのgoalの達成条件から外す。
